Training Workflow

Model training runs offline via a command-line script. This page handles steps 1–2; steps 3–5 use the Model Evaluation page.

  1. Label detections — use the feedback controls on the Events page to mark each detection as Correct, False Positive, or Wrong Class.
  2. Export the dataset — click Download Dataset above to get a YOLO-format zip (images + annotation .txt files + data.yaml).
  3. Train a model — run training/train.py on a GPU machine (the Orin works, but a workstation GPU is faster for training): 
    python train.py --data dataset/data.yaml --output heron-v2.pt --epochs 100
    See training/README.md in the repo for the full CLI reference and Jetson tips.
  4. Upload the model — drop the resulting .pt file into the Models page.
  5. Evaluate & promote — compare the new model against the current one on the Model Evaluation page, then promote the winner. The detector hot-reloads the new model — no restart needed.
